Comparison

RAG vs Wiki

Two approaches to giving an LLM domain knowledge. One compounds. One doesn't.

RAG — Retrieve then Think

  • ×Every query re-derives from scratch
  • ×Cross-refs need all sources in retrieval window
  • ×Contradictions invisible unless you ask
  • ×Yesterday's analysis is gone tomorrow
  • ×Scales via embedding similarity

Wiki — Think once, Retrieve many

  • ✓Knowledge compiled once at ingest time
  • ✓Cross-references already built and maintained
  • ✓Contradictions flagged automatically on ingest
  • ✓Syntheses persist and accumulate
  • ✓Scales via structured pages + index

RAG: Retrieve-then-Think

  • ×Re-derives every time. Ask "what plans use threshold cliffs?" today and tomorrow — the LLM does the same work twice. No memory of prior analysis.
  • ×Retrieval window is the bottleneck. If a cross-reference needs 5 source chunks to be visible, you need all 5 in the top-k. Miss one and the connection is invisible.
  • ×Contradictions are silent. If Source A says accelerators go up and Source B says they can go down, RAG won't flag it unless you explicitly ask "are there contradictions?"
  • ×Clusters by text similarity, not mechanics. "Medical device territory manager" clusters with medical device content. "SaaS CSM" clusters with SaaS content. The shared threshold-cliff mechanic is invisible because the text is different.
  • ×Good at breadth, bad at depth. Works well for "find me something about X." Fails at "what have we learned about X across all plans we've ever analyzed?"

Wiki: Think-once-Retrieve-many

  • ✓Compiled at ingest. When a plan enters the system, the LLM reads every existing concept page, checks for overlaps and contradictions, creates new pages, and updates confidence scores. That work is done once.
  • ✓Cross-references are pre-built. The "Threshold-Cliff" page already links to MedCo (70% floor) and SaaSPivot (85% floor). No retrieval window needed — the connection is a [[wikilink]].
  • ✓Contradictions are first-class. Every concept page has a "Contradictions" section. When MedCo's decelerator contradicted Acme's accelerator-only model, it was flagged during ingest — not discovered by accident.
  • ✓Confidence accumulates. Hard-Cap-Ceiling started at 0.70 after one source. Now it's 0.90 after four sources confirmed the same pattern. The number tells you how much evidence backs each claim.
  • ✓Syntheses persist. The 17-gap IR converter analysis was done once. It lives as a synthesis page. Every future session can read it instead of re-deriving it.

Key Discoveries

Where Compilation Beat Retrieval

Three real examples from our wiki where pre-compiled knowledge found things RAG would have missed.

1
Cross-Industry Pattern Discovery
Medical devices and SaaS share a hidden mechanic
▾

MedCo (medical device Territory Manager) has a 70% threshold floor. SaaSPivot (SaaS CSM) has an 85% threshold floor. Different industries, different roles, different plan shapes.

But they share the same mechanic: Threshold-Cliff — a hard floor below which variable payout is zero.

Why RAG misses this: RAG clusters by text similarity. "Medical device territory manager" groups with med-device content. "SaaS customer success" groups with SaaS content. The mechanical similarity is invisible because the words are different.

Why the wiki caught it: During ingest #4, the LLM read the existing Threshold-Cliff concept page (created for MedCo) and recognized SaaSPivot's 85% cliff as the same mechanic. It added a cross-industry comparison table automatically.

Discovered at ingest time, not query time
2
Schema Ambiguity Detection
The same JSON array means two opposite things
▾

Acme Corp's plan uses an accelerators[] array with multipliers above 1.0 — standard upward-step acceleration.

MedCo's plan uses the same accelerators[] array but with a multiplier of 0.75 at the 80% threshold. That's not acceleration — it's a decelerator.

Why RAG misses this: RAG treats each chunk independently. It would never compare the schema shape of Acme's accelerators against MedCo's unless both happened to land in the same retrieval window for the same query.

Why the wiki caught it: When MedCo was ingested, the LLM updated the existing "Tiered-Accelerator-Plans" concept page. It noticed the sub-1.0 multiplier contradicted the page's upward-only assumption and added a Contradictions section. This directly informed SCD's schema design.

Contradiction flagged automatically during ingest
3
The 17-Gap Blueprint
Wiki concepts mapped directly to missing code
▾

SCD's IR converter emits 3 behaviors. A production-grade comp plan (TechCorp reference) needs 22 behaviors. That's 14% coverage.

The wiki synthesized a gap analysis by mapping compiled concept pages to missing IR behaviors:

Event-Driven-Release (wiki concept) → 4 missing behaviors
Management-Overrides (wiki concept) → 2 missing behaviors
Recovery-Precedence (wiki concept) → 4 behaviors + 1 command
Rate/Attainment chain (wiki concept) → 3 missing behaviors
Rollups + bonuses (wiki concepts) → 3 behaviors + 1 command

Why RAG can't do this: This synthesis required reading 8+ wiki concept pages, a reference IR file, and the actual converter source code — then cross-referencing them into a structured gap analysis. RAG would have to re-derive this from scratch every time someone asked.

Why the wiki makes it permanent: The synthesis lives as a page. Every future session reads it. The converter rebuild references it directly.

17 gaps identified, 5 work clusters defined, referenced by active dev work
4
First Real Client Plan Validates the Wiki
Fortune 500 medical device plan confirmed patterns AND discovered 4 new mechanics
▾

We ingested a real Medtronic FY25 District Manager SIP — 9 measures, $195K-$230K annual target, 20 pages. The wiki had only seen synthetic plans before this.

What the wiki validated: The 70% threshold cliff matched MedCo (synthetic). Hard-cap language was the strongest seen across 5 sources. Confidence scores went UP — Hard-Cap-Ceiling hit 0.95, the highest in the wiki.

What the wiki discovered (NEW mechanics):

Inverse Expense Bonus — lower spending = higher payout. Breaks the assumption that all measures reward higher attainment.
Administrative Eligibility Gate — implant registration deadline excludes transactions from payout. First non-financial gate.
Year-End True-Up Guarantee — annual payout floor for quarterly volatility protection.
Product Segmentation Strategy — innovation carve-out (Micra+Aurora) separated from parent categories to steer focus.

Why this matters: The wiki went from "interesting prototype with synthetic data" to "validated knowledge system that handles real Fortune 500 complexity" in a single ingest.

4 new concepts, 2 confidence upgrades, 12 pages touched in one ingest

Cross-Plan Discoveries

Within the same medical device company across 6 BUs, threshold varies systematically: 50% → 60% → 70% → 85%. Draw type correlates with seniority: non-recoverable at $250K, recoverable at $20K-$230K, none below that. These patterns only appear with compilation.

The compounding effect: Plan #1 had zero cross-references. Plan #14 has 100+ cross-references and 33 concept pages. Plan #35 (when compiled) will push us past 50 concepts with confidence scores backed by cross-industry evidence. Every new client plan makes the system smarter. That's what RAG can't do.

Every new client plan makes the wiki smarter. After 5 plans, the wiki knows 18 distinct compensation mechanics, their schema expressions, and where they contradict each other. Plan #6 benefits from everything learned in plans 1-5.

Grounded Extraction

SCD's LLM extractor can use compiled wiki concepts as prompt grounding instead of relying on training data. "Here are the 18 known mechanics with definitions" is better than "figure out what mechanics this plan uses."
