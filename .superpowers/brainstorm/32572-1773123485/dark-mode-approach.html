Dark Mode Strategy for PROOFLINE Demo

The demo currently uses hardcoded light colors (#1A1A2E text, bg-white cards, #718096 muted). How should dark mode work?

A

Theme-Aware CSS Variables (Recommended)

Add a dark class toggle to DemoShell. All 30 pages already use inline styles with hardcoded hex colors — replace those with CSS custom properties that flip between light/dark values. The toggle lives in the DemoShell header bar next to the "Interactive Demo" badge.

Pros

  • One toggle controls everything
  • Matches the iPad PROOFLINE demo pattern
  • CSS vars cascade automatically

Cons

  • Must touch all ~30 pages to replace inline styles
  • Barrel-brass theme preset needs a dark companion
B

Dual Theme Presets

Create a barrel-brass-dark theme preset alongside the existing barrel-brass. The DemoShell toggles between presets, swapping all CSS variables at once. Pages don't change at all — but they still have hardcoded inline colors that won't respond to the swap.

Pros

  • Clean theme architecture
  • Minimal DemoShell changes

Cons

  • Still must fix all 30 pages' inline styles
  • Two presets to maintain in sync
C

CSS filter: invert() Quick Hack

Apply a CSS filter to invert colors on the content area. Fast to implement, zero page changes needed. But inverted colors look awful on charts, SVGs, and branded elements.

Pros

  • Zero page changes
  • 5-minute implementation

Cons

  • Charts and SVGs look broken
  • Brand colors get inverted
  • Unprofessional appearance
