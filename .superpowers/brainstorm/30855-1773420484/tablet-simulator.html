POS Tablet Simulator — Launch Pattern

How should the POS terminal appear when demoing? Pick the approach that works best for presenting to clients.

A

Popup Window (ROUTEIQ Pattern)

Manager Console has a "Open on iPad" button that launches window.open() with specific dimensions (1024x768). The POS opens in a separate browser window styled as a tablet with a CSS bezel frame (camera notch, home indicator, status bar). You position it next to the main browser during the demo.

Pros

  • Two windows side-by-side = manager + rep tablet visible simultaneously
  • BroadcastChannel works perfectly (same origin)
  • Most realistic — feels like a separate device
  • Proven pattern from ROUTEIQ demo

Cons

  • Popup blockers may interfere
  • Requires positioning two windows manually
B

Slide-Over Panel

The POS terminal slides in from the right as a full-height overlay panel (480px wide) on top of the current page. Has an iPad bezel frame. Manager page stays visible behind it (dimmed). Close button to dismiss.

Pros

  • No popup blocker issues
  • Single window — easier to present on a projector
  • Smooth animation, feels polished

Cons

  • Cramped at 480px — POS UI needs to be compact
  • Can't see manager + POS side by side (overlay hides manager)
  • Less realistic as a "separate device"
C

Dedicated Route in TabletFrame

POS lives at its own URL (/register/ops/pos-terminal) wrapped in ROUTEIQ's TabletFrame component — a CSS iPad bezel with camera notch, status bar, and home indicator. Navigate to it directly or via a button that opens a new tab. Full-width tablet experience.

Pros

  • Full screen real estate for the POS UI
  • Reuses existing TabletFrame component from ROUTEIQ
  • Can open in new tab for side-by-side with manager
  • Works great on actual iPads too

Cons

  • Navigating away from manager loses context (unless new tab)
  • Need "Open on iPad" button that opens new tab

Recommendation: Option A or C

For client presentations, the popup window (A) gives you the most dramatic "push to iPad" moment — you click a button on the manager screen and a tablet literally appears. Option C is the most pragmatic and reuses existing code. Both support the BroadcastChannel real-time push story.
